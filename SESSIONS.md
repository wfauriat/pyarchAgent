### 2026-05-27 ‚Äî `chat()` error model: transport + protocol layers

- **Working mode**: deliberated up-front on whether to do "Claude writes broken code, user PR-reviews it" vs tutor-first. Settled on tutor-first now, PR-review exercise reserved for a later function (conversation history or tool dispatcher) so design-and-write reps come first.
- **Three-layer failure taxonomy** adopted as the mental model for the HTTP call: **transport** (couldn't reach server ‚Äî `httpx.RequestError` family), **protocol** (non-2xx status code ‚Äî `httpx.HTTPStatusError`), **contract** (response received but JSON shape wrong ‚Äî `KeyError` / `JSONDecodeError`). Each layer has its own moment in the call sequence and its own remediation.
- **Design call**: transport = *runtime* concern, surface as catchable exception so caller can retry/alert. Protocol + contract = usually *programmer errors*, fail loud with good context. Don't catch what you can't sensibly handle.
- **Return-type honesty**: identified the old pattern (`except BaseException: return str(e)`) as a lying return type ‚Äî caller can't distinguish a real reply from an error string. Replaced with raise-based error model. `-> str` now actually means "the assistant's reply, or this function raises."
- **Custom exceptions, not pass-through**: introduced `BackendConnectionError` and `BackendResponseError` as module-level classes. Chosen over (a) bare `raise` of httpx exceptions and (b) builtin `ConnectionError` ‚Äî reason: project explicitly plans a second backend (Anthropic), and both adapters should raise the same application-level types so caller code stays backend-agnostic from day one. Common `BackendError` base class noted but deferred.
- **Concrete idioms picked up**: `response.raise_for_status()` to convert non-2xx to exception; `raise X(...) from e` to wrap while preserving the original via `__cause__`; implicit string concatenation inside parens for multi-line f-string messages (no `+`, no `\`).
- **httpx hierarchy insight**: `RequestError` and `HTTPStatusError` are *siblings* under `HTTPError`, not parent/child ‚Äî so a single try block with two except clauses suffices, no nested try needed. Current code still has the nested form; flatten on next pass.
- **Empirical confirmations**: happy path returns model reply; bogus-port test produces `BackendConnectionError` with chained `httpx.ConnectError` underneath. Protocol path not yet exercised against a live Ollama (needs real port + nonexistent model name).
- **Side**: added `editor.rulers: [80]` to `.vscode/settings.json` for a soft 80-col guide. Brief detour on Python multi-line string idioms.
- **Next**: (1) flatten the nested try block; (2) fix the misleading `BackendResponseError` message to include `e.response.status_code` and `e.response.text`; (3) provoke and confirm the protocol path empirically; (4) decide whether to add a `BackendError` common base; (5) then move on to multi-turn conversation history.

### 2026-05-27 ‚Äî `chat()` design audit, parameterization pass, rename to `ollama_backend.py`

- **Design audit of `chat()` as a module-level API** (vs script-level function). Identified five things that would matter once the function gets a second caller or a second backend: (1) module-level constants read implicitly, (2) `httpx.post` creates+discards a client per call (no DI seam, no connection reuse), (3) `"think": False` hardcoded in the payload (Ollama- and qwen3-specific), (4) return type is bare `str` ‚Äî soon insufficient for token counts / stop reasons / tool calls, (5) no abstraction over backend choice.
- **Applied (1)‚Äì(3) one step at a time** so each change was reviewable in isolation:
  - **Parameterize config** as keyword-only args with module-constant defaults: `model`, `url`, `timeout`. Use of the `*,` separator forces keyword calls at the override site ‚Äî readable when there are 3+ optional args.
  - **Inject `httpx.Client`** via `client: httpx.Client | None = None`. Default `None` falls back to `httpx.post()` (today's behavior); passing a client enables connection reuse, shared headers (auth), and test-time injection of fakes. Avoided `client = httpx.Client()` as default ‚Äî that's the mutable-default-at-import footgun.
  - **Expose `think`** as a keyword arg. Was hardcoded; now per-call.
  - Default-only call site (`chat(messages)`) is unchanged ‚Äî `__main__` block needed no edits. Good shape for API evolution.
- **Parked (4) ‚Äî structured return type.** Promoting to `@dataclass ChatResult { content, tokens_in, tokens_out, stop_reason, ... }` is the right move *when* a caller actually needs more than the assistant string. Doing it now is speculative. Note: widening this return type later is a breaking change for any caller using `chat(...)` directly ‚Äî accept that future churn, or front-load when the first real need lands.
- **Parked (5) ‚Äî backend abstraction (ABC / Protocol / class refactor).** Sibling per-backend files (`ollama_backend.py`, future `anthropic_backend.py`) sharing the `BackendError` hierarchy is the working pattern for now. An abstract `Backend` ABC or `Protocol` becomes worthwhile the moment there are two adapters *and* a caller that wants to swap between them generically. Until then, "caller picks the import" is cleaner than an inheritance tree built for one occupant.

### 2026-05-29 ‚Äî Ollama error-path tests, `pytest.raises`

- **Two error-translation tests added** for `chat`: `post` raising `httpx.RequestError` ‚Üí `BackendConnectionError`, and `raise_for_status` raising `httpx.HTTPStatusError` (real `Request`/`Response` attached) ‚Üí `BackendResponseError`. - **Error-translation testing pattern banked:** the *fake* raises the real low-level library error; the *unit under test* catches and re-raises its own; the *test* asserts the unit's error ‚Äî and asserting the message proves the unit extracted fields (status code/text) off the original, not just that *something* raised.

### 2026-05-29 ‚Äî Interface decision: wide contract + `Backend` Protocol, mypy verification

### 2026-05-30 ‚Äî Follow-up: Pyright invariance on the nested client Protocol

- **The `FakeClient` warning persisted in the editor after the first fix** even though `mypy --check-untyped-defs` was clean. ### 2026-06-03 ‚Äî Tool-execution thread opened: `run_bash` subprocess scratch + bash craft

- **The dormant bash craft finally wakes.** Opened the tool-*execution* thread with a throwaway `scratch_subprocess.py` (untracked ‚Äî observe-before-abstract, like the project's first move): `run_bash(command) -> BashResult` over `subprocess.run(["bash", "-c", cmd], capture_output=True, text=True, timeout=10, check=False)`.
- **Two deliberate calls in the `run` invocation.** (1) `["bash", "-c", cmd]` + `shell=False` gets *bash* semantics on purpose ‚Äî `shell=True` would use `/bin/sh` (dash on Ubuntu), losing `[[ ]]`/process-substitution. (2) `check=False` because a non-zero exit is **information to report to the model**, not a Python error ‚Äî no `CalledProcessError`.
- **`BashResult` frozen dataclass** (`returncode`, `stdout`, `stderr`, `timeout: bool = False`). Split "timed out" into its own field rather than overloading `returncode` (illegal-states-unrepresentable instinct, same as `ChatResult`/`Message`). Timeout branch returns `-9` (honest SIGKILL) **and** `timeout=True` ‚Äî redundant but both true; the loop should branch on the flag, not the magic number.
- **Empirical gotcha #1 ‚Äî `text=True` does NOT decode the partial output on timeout.** Running `printf '...'; sleep 20` showed `e.stdout` returns raw **bytes** (`b'partial line\n'`) and `e.stderr` is `None`, so a naive `str(e.stdout)` would leak a `b'...'` repr / the literal `"None"` into the model context. Built `_normalize(x: bytes | str | None) -> str` to fix it.
- **`match` lesson ‚Äî capture vs class patterns.** First `_normalize` used `case bytes:` ‚Äî but a bare name is a *capture pattern* (binds the name, matches anything), not a type test; Python **refuses to compile** it ("name capture 'bytes' makes remaining patterns unreachable"). Fix is class patterns `case bytes():`. Then switched to a plain `isinstance` ladder ‚Äî `match` earns its keep for *destructuring*, not flat type-dispatch.
- **Empirical gotcha #2 (Pylance) ‚Äî the bare-`bytes` promotion.** A `bytes` annotation is implicitly promoted to `bytes | bytearray | memoryview` (buffer-protocol promotion; PEP 688 backstory; `typing.ByteString` is the deprecated explicit form ‚Äî PEP 594, gone in 3.14). So `isinstance(x, bytes)` *negative* narrowing leaves `bytearray | memoryview`, and reaching `str` by **elimination** fails the `-> str`. Fix: confirm `str` **positively** (`isinstance(x, str): return x`), decode the buffer family via `bytes(x).decode(errors="replace")` (`memoryview` has no `.decode`, hence the `bytes(x)` wrap). **Principle banked: return type T by positively confirming T, not by eliminating everything else** ‚Äî robust to promotions and future union growth.
- **Mapped the executor gotcha landscape** (stdin hang, output blowup, orphan/process-group on timeout, cwd/env statelessness across calls, decode errors) and triaged: **add `stdin=subprocess.DEVNULL` now** (one kwarg ‚Äî prevents the indefinite hang the moment a stdin-reader like `cat`/`grep` runs); **defer output truncation to the wiring/render step** (it's a property of the model-facing string, not `run_bash`); **park** process-group killing (needs `Popen`) and the persistent-shell-vs-stateless-`cwd` fork (belongs *in* the concrete-tools design).
- **Checkpoint discipline before destructive work.** Committed the lingering **Mistral backend** + **Makefile help/`.PHONY` sync**, and folded the **runner wiring** (`__init__` export + `-b mistral` choice + `elif` branch ‚Äî `make chat_mistral` now works) into the Mistral commit, so the tree is clean before the agent starts running real bash. Convention reaffirmed: **no `Co-Authored-By` trailer** on commits (user authors most code).
